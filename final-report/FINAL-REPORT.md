# Final Report on Tech Warmup II: AI-Assisted Slot Machine Development

## Introduction
Our team used Gemini-3-flash-preview through the Gemini CLI as our primary AI model. We selected this setup due to its accessibility and ease of use across all team members.

Over 30+ iterations, we developed a slot machine web application starting from a simple random number generator and gradually adding features such as multiple game modes, animations, sound effects, and unit tests.

## Development Process
We followed an iterative, AI-assisted workflow:

1. Break features into small, manageable tasks
2. Prompt AI to generate initial implementations
3. Test the generated code
4. Identify issues or incorrect behavior
5. Refine prompts and repeat

This loop was repeated for each feature, allowing us to gradually improve both functionality and code quality.

## Prompt Refinement
One of the most important aspects of our process was improving how we wrote prompts.

Early prompts were often vague, which led to incorrect or incomplete implementations. Over time, we learned to:
- Specify expected inputs and outputs
- Include edge cases
- Request more structured and modular code

This significantly improved the quality and reliability of AI-generated code.

## Challenges
While AI was helpful, it introduced several challenges:

- AI often misinterpreted requirements
- Generated code sometimes had logical errors despite compiling correctly
- Edge cases were frequently missing
- Code quality and structure were inconsistent

Additionally, we encountered traditional software issues such as:
> “Works on my machine but not others”

This highlighted the importance of testing and environment consistency.

## Key Learnings
Through this project, we learned:

- AI is most effective when used in small, incremental steps
- Prompt quality directly impacts output quality
- AI cannot replace debugging and validation
- Tools like linters help improve consistency
- Careful planning improves the effectiveness of AI

## Final Reflection
AI significantly accelerated development and reduced the time needed to write initial implementations. However, it does not replace core software engineering skills such as planning, debugging, and testing.

Ultimately, we found that AI works best as a powerful assistant rather than an autonomous developer.
